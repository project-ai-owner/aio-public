**2026-07-11**

このサイトの最初の記事として、公開に至る判断そのものを記録します。

## 転換前の方針

AIO は 2026年6月末の開始以来、非公開の仕組み・事業候補として育ててきました。AI エージェントの役割分担、GitHub を単一の正とする運用、権限層による自動化統制 — こうした構成には希少性があり、秘匿する価値があると考えていました。

## 何が変わったか

約 2 週間、毎日運用して分かったのは、**個別の要素技術はすでに希少ではない**ということです。AI エージェントの多重化も、スケジューラ実行も、PR ベースの変更管理も、先行して AI を活用している人にとっては手の届く道具になっています。

一方で、希少性が残っている（そして検証可能な形で示せる）のは、**制約下で実際に運用し、失敗し、設計を直した過程そのもの**でした。

- メモリ 2GB の VM と AI 利用上限という現実的な制約
- 2 つの AI に相互チェックさせて破綻した記録
- 自動化を急ぎすぎて停止スイッチを常置した記録
- 並列エージェントの PR 競合を構造で解いた記録
- それらすべてが Decision・PR・改善項目として GitHub に残っていること

つまり、AIO の価値は「仕組みの秘密」ではなく「**設計・運用・失敗分析の能力の証明**」にある。それなら公開した方が価値が高い、という判断です。

## 公開の設計

公開自体も AIO の権限モデルに従って設計しました。

- **実運用リポジトリは公開しない**。Git 履歴や内部文書には機密が混入しうるため、外部向けに再構成した文書だけを置く公開専用リポジトリ（このリポジトリ）を新設
- **公開は段階制**。記事はすべて PR 経由で追加し、公開前チェックリスト（機密・誇張・現状一致・公開可否・読者価値）と CI の機械チェック（secret スキャン＋禁止パターン検査）を通す。サイトの一般公開そのものも Human の承認事項
- **執筆体制も AIO 流**。この記事を含むサイトの実装・起案は Claude Code が行い、Human が公開前に確認する。将来的には非公開側の変更履歴から公開候補記事を自動起案する構成を目指すが、**完全自動公開はしない**

## 今後

まず基本ページ（Overview / Architecture / Lessons / Current Status）を揃えて一般公開し、その後は開発ログとして設計変更・失敗分析・コスト分析を継続的に書いていきます。未完成のものを完成済みと書かない、が このサイトの唯一の絶対ルールです。
