## これは何か

**AI Improvement OS（AIO）** は、AI エージェント（Claude Code）と人間が役割分担しながら、プロジェクトの改善サイクル（課題発見 → 意思決定 → 実装 → レビュー → 記録）を回し続けるための運用基盤です。2026年6月28日に開始し、この記録の執筆時点で **コミット 716 件・意思決定記録（Decision）45 件・改善項目（IMP）143 件** を GitHub 上に積み上げています。

## なぜ作ったのか

「AI にタスクを頼む」ことと「AI に運用を任せられる状態を作る」ことの間には大きな距離があります。権限の境界・変更管理・並列実行の衝突・コンテキストの限界・コストの管理 — この距離を埋める仕組みを、実際に手を動かして確かめるために作っています。

## 誰が、どのような構成で作っているのか

IT インフラの運用・保守を本業とする 30 代の会社員が、個人活動として構築しています（実名は公開していません）。業務ではシステムを安定して運用するための監視・障害対応・変更管理・手順整備などに携わってきましたが、**AI や機械学習を専門とする職種ではありません**。本業と育児を並行しながら、限られた時間と予算の中で運用しています。

AIO は AI 研究の成果を示すプロジェクトではなく、**既存の運用経験を生かし、一般に利用できる AI サービスを現実の仕事へどう組み込めるかを検証する**プロジェクトです。環境はパイロット段階の初期構成として意図的に小さく始め、実測に応じて拡張する方針です。

- **計算資源**: AWS EC2 t3.small 1台（メモリ 2GB）から開始。並列エージェント数は当面 2 に設定。現時点ではこのスペックで運用が回っており、必要になった時点で拡張する
- **AI 利用枠**: Claude Pro の利用枠内で運用（モデルを用途別に階層化して配分する規律）
- **時間**: 人間が常時張り付けない前提。バックグラウンド実行と非同期レビューが必須
- **コスト規律**: サブスクリプション＋従量課金を明示的にガード（例: 外部 API に月次上限を設定）。実測に連動して段階拡張する

## 現在どこまで動いているのか

日次の定期ルーチン（リポジトリ監査・外部 AI 出力の取り込み・コンテンツ準備）と、X（SNS）への定時自動投稿パイプラインが**実際に無人で動いています**。一方で、タスクキューの完全自動化は一度作った後に**人間の判断で停止中**です。動いているもの・止めたもの・未完成なものの区分は [Current Status](./status/) に正直に書いています。
