## これは何か

**AI Improvement OS（AIO）** は、AI エージェント（Claude Code）と人間が役割分担しながら、プロジェクトの改善サイクル（課題発見 → 意思決定 → 実装 → レビュー → 記録）を回し続けるための運用基盤です。2026年6月28日に開始し、この記録の執筆時点で **コミット 716 件・意思決定記録（Decision）45 件・改善項目（IMP）143 件** を GitHub 上に積み上げています。

## なぜ作ったのか

「AI にタスクを頼む」ことと「AI に運用を任せられる状態を作る」ことの間には大きな距離があります。権限の境界・変更管理・並列実行の衝突・コンテキストの限界・コストの管理 — この距離を埋める仕組みを、実際に手を動かして確かめるために作っています。

## 誰が、どのような制約下で作っているのか

会社員としての本業を持つ個人が、業務外の時間で運用しています（実名は公開していません）。主な制約は次のとおりです。

- **計算資源**: AWS EC2 t3.small 1台（メモリ 2GB）。並列エージェント数は 2 に制限
- **AI 利用上限**: Claude Pro の利用枠内で運用（モデルを用途別に使い分けて節約）
- **時間**: 人間が常時張り付けない前提。バックグラウンド実行と非同期レビューが必須
- **コスト**: サブスクリプション＋従量課金を明示的にガード（例: 外部 API に月次上限を設定）

## 現在どこまで動いているのか

日次の定期ルーチン（リポジトリ監査・外部 AI 出力の取り込み・コンテンツ準備）と、外部サービスへの定時自動実行が**実際に無人で動いています**。一方で、タスクキューの完全自動化は一度作った後に**人間の判断で停止中**です。動いているもの・止めたもの・未完成なものの区分は [Current Status](./status/) に正直に書いています。
