## 実際に動いているもの

- **GitHub SSOT 運用**: 全変更が commit / PR / merge 経由。コミット 716 件・Decision 45 件・IMP 143 件（2026-06-28 開始から約 2 週間の実績）
- **クラウド定期ルーチン**: 日次リポジトリ監査（バックログ整合・PR 滞留・文書ライフサイクルの機械チェック）／外部 AI 出力の取り込み（6時間毎・冪等）／事業ケース向けコンテンツ準備（日次・週次・月次）
- **VM 定時実行**: X への 1 日 1 件の自動投稿（多重実行防止・停止スイッチ・独立ファクトレビュー合格コンテンツのみ投稿・月次予算ガード付き。[Architecture の適用ケース](../architecture/) 参照）
- **PR 作成前セルフレビュー（Review Agent）**: 8 項目チェックを通過した PR のみ作成
- **権限 3 層モデルと merge 代行**: 条件を満たす PR は Claude が merge を代行。ガバナンス変更は「Claude が説明 → Human が承認 → Claude が merge」
- **subagents（4種）と権限ガード hook**: コンテキスト分離委任と、危険操作の機械的ブロック

## 一部だけ動いているもの

- **X 運用のフィードバック計測ループ**: 分析・振り返りの生成は自動化済みだが、**計測データの回収は人間の手作業**（週 1 回のスクリーンショット共有）に依存
- **外部 AI（ChatGPT）連携**: インボックス経由の取り込みは自動だが、依頼の受け渡しは人間が仲介
- **トークン・コスト計測**: 定期ルーチンがメトリクスを記録しているが、バックグラウンドエージェントのトークン数など取得できない項目が残る
