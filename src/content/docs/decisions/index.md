AIO では、承認スコープ・運用フロー・技術選定・ガバナンスに関わる変更をすべて **Decision** として記録しています（非公開リポジトリに 45 件・2026-07-11 時点）。ここではそのうち、仕組みの設計として意味のあるものを簡略 ADR 形式で順次公開します。

## 公開予定の Decision（順次追加）

| # | 決定 | 一言要約 | 状態 |
|---|---|---|---|
| ADR-01 | GitHub を SSOT とする | セッションが揮発する AI の耐久記憶を PR・記録ファイルに置く | 有効 |
| ADR-02 | 最終意思決定は Human | AI の merge 代行は権限層の条件付き。停止権限は常に Human | 有効 |
| ADR-03 | トークン効率化原則 | 読み方・書き方・セッション管理の規律で AI 利用枠を節約 | 有効 |
| ADR-04 | 実行環境を VM に固定 | ファイル操作・git 操作は常時稼働 VM 上のみ。ローカル PC へのフォールバック禁止 | 有効 |
| ADR-05 | 権限 3 層モデル | 事前承認済み / 条件付き（Human 許可）/ 非解除の禁止、の 3 層で自動化可否を判定 | 有効 |
| ADR-06 | 単一エージェント統合 | ChatGPT の merge 承認役を廃止し Claude に統合（[Lessons 01](../lessons/01-two-agent-retirement/)） | 有効 |
| ADR-07 | Chat-Primary + Background-First | Human との窓口は単一チャット・作業はバックグラウンド優先 | 有効 |
| ADR-08 | 並列実行の競合回避 | 担当ファイルの disjoint 検証と共有インデックスの直列化（[Lessons 03](../lessons/03-shared-index-conflicts/)） | 有効 |
| ADR-09 | 文書体系の再設計 | 正本 1 箇所原則・アーカイブ規約・ホットファイルの世代分割 | 有効 |
| ADR-10 | プロジェクトレベル subagents | コンテキストスコープで作業系を、ロールで安全系を分割した 4 subagents | 有効 |
| ADR-11 | 実践記録の一般公開 | 本サイトの新設。公開専用リポジトリ分離・公開前チェック・段階的公開 | 有効（本サイトの根拠） |

各 ADR の詳細ページは順次追加します。「決定したが後に変更・廃止したもの」（merge リスク判定モデルの改訂、外部実行方式の変更など）も、変更理由つきで公開する方針です。
