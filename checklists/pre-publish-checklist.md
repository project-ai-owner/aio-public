# 公開前チェックリスト

すべての公開コンテンツ PR で、merge 前に以下 5 点を確認する（正本: 非公開リポジトリの D-045）。

1. **機密情報が含まれていないか**
   - secret / token / API キー / 鍵・認証情報のパス内容
   - VM のパブリック IP・内部ホスト名・接続情報
   - 個人メールアドレス・実名・家族・勤務先の非公開情報
   - trigger ID・内部 ID 類
   - 検討中ビジネスケースの内容・KPI・収支・運転資金額
2. **誇張や事実誤認がないか**（数値・日付・構成は実物で裏取りされているか。推定は推定と明記されているか）
3. **現在の実装状況と一致しているか**（廃止済み・停止中のものを「動作中」と書いていないか。Current Status の区分と矛盾しないか）
4. **外部公開して問題ないか**（利用規約・ライセンス・第三者の権利・攻撃への転用可能性）
5. **読者にとって意味のある内容になっているか**（宣伝表現でなく、具体的な制約・構成・失敗・改善が書かれているか）

機械チェック（CI）: `scripts/publish-guard.sh`（禁止パターン検査）＋ gitleaks（secret スキャン）。機械チェックは補助であり、本チェックリストの目視確認を置き換えない。
